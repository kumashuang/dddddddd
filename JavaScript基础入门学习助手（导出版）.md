# 模块一：变量声明

## 一、知识点总结

- **变量的作用**：用来存储数据（如数字、文字、布尔值等），方便后续使用和修改。

- **3种声明方式**：
        

    - let：最常用，声明可修改的变量，有块级作用域（\{\}内有效），不能重复声明。

    - const：声明不可修改的常量（值固定），有块级作用域，必须初始化（声明时赋值），不能重复声明。

    - var：旧语法，无块级作用域，可重复声明、可提升，现在不推荐使用（了解即可）。

- **命名规则**：
       

    - 只能由 字母、数字、下划线（\_）、美元符号（$） 组成。

    - 不能以数字开头。

    - 不能使用JS关键字（如let、const、var、if、for等）。

    - 区分大小写（如userName和username是两个不同变量）。

    - 推荐使用「驼峰命名法」：首字母小写，后面单词首字母大写（如userAge、studentName）。

- **核心注意点**：let和const是ES6新增语法，解决了var的缺陷，日常开发优先用let（需修改）和const（不修改）。

## 二、练习题（3道，含详细解析）

### 练习题1：判断下列变量声明是否正确，若错误请说明原因

```js
// 题目
1. let 123name = "张三";
2. const PI = 3.14159;
3. let userName = "李四";
4. var userName = "王五";
```

#### 答案：

- 错误：变量名不能以数字开头（123name不符合命名规则）。

- 正确：const声明常量，初始化赋值，命名符合规则。

- 正确：let声明变量，驼峰命名，符合规则。

- 错误：let和var声明了同名变量（userName），重复声明（即使let和var混用，也不允许重复声明同一变量）。

#### 解析：

重点考查「变量命名规则」和「重复声明禁忌」：数字不能开头，let/const/var之间不能重复声明同一变量；const必须初始化，这里PI赋值正确，符合要求。

### 练习题2：写出下列代码的运行结果，并说明原因

```js
// 题目
let age = 18;
age = 20;
const gender = "男";
gender = "女";
console.log(age);

```

#### 答案：

运行报错：Uncaught TypeError: Assignment to constant variable\.（无法给常量赋值）

若删除「gender = \&\#34;女\&\#34;」这一行，运行结果为：20 、 男

#### 解析：

- const声明的常量（gender）不能修改，试图给gender赋值\&\#34;女\&\#34;，会触发语法错误，导致代码无法继续运行。

### 练习题3：用正确的方式声明以下变量/常量

```js
// 题目
1. 声明一个变量，存储用户姓名，初始值为"张三"，后续可能修改
2. 声明一个常量，存储圆周率，值为3.14159
3. 声明一个变量，存储用户年龄，暂时不赋值，后续赋值
```

#### 答案：

```js
// 1. 可修改，用let
let userName = "张三";

// 2. 不可修改，用const
const PI = 3.14159;

// 3. 暂时不赋值，用let（const必须初始化，不能不赋值）
let userAge;
```

#### 解析：

核心考查「let和const的使用场景」：需要修改的变量用let，不需要修改的常量用const；let可以先声明不赋值，const必须声明时就赋值，否则报错。

# 模块二：数据类型

## 一、知识点总结

JavaScript的数据类型分为「基本数据类型」和「引用数据类型」，零基础先重点掌握基本数据类型。

- **基本数据类型（5种，重点掌握）**：
        

    - Number（数字型）：包含整数、小数、正数、负数、NaN（非数字，如\&\#34;abc\&\#34;转数字时得到）。例：10、3\.14、\-5、NaN。

    - String（字符串型）：用单引号（\&\#39;\&\#39;）、双引号（\&\#34;\&\#34;）包裹的文本。例：\&\#34;hello\&\#34;、\&\#39;JS入门\&\#39;、\&\#34;\&\#34;（空字符串）。

    - Boolean（布尔型）：只有两个值，true（真）和false（假），常用于判断条件。例：true、false。

    - Undefined（未定义型）：变量声明了，但没有赋值，默认值是undefined。例：let a; 此时a的值是undefined。

    - Null（空值型）：表示“空”，主动赋值为null，常用于表示一个空对象或空值。例：let b = null;。

- **引用数据类型（了解即可，后续深入）**：对象（\{\}）、数组（\[\]）、函数（function）等。

- **查看数据类型的方法**：typeof 变量名，返回变量的数据类型（注意：typeof null 会返回\&\#34;object\&\#34;，是JS的小bug，记住即可）。

- **核心注意点**：
        

    - 字符串必须用引号包裹，否则会被当作变量（报错）。

    - NaN是Number类型，但它不等于任何值，包括它自己（NaN === NaN 结果是false）。

    - undefined和null的区别：undefined是“未赋值”，null是“主动赋值为空”。

## 二、练习题（3道，含详细解析）

### 练习题1：判断下列变量的数据类型，写出typeof的返回结果

```js
// 题目
let a = 123;
let b = "123";
let c = true;
let d;
let e = null;
```

#### 答案：

- typeof a → \&\#34;number\&\#34;（a是数字型）

- typeof b → \&\#34;string\&\#34;（b是字符串型，即使内容是数字，用引号包裹就是字符串）

- typeof c → \&\#34;boolean\&\#34;（c是布尔型）

- typeof d → \&\#34;undefined\&\#34;（d声明未赋值，默认是undefined）

- typeof e → \&\#34;object\&\#34;（JS bug，null是Null类型，但typeof返回\&\#34;object\&\#34;）

#### 解析：

重点考查「基本数据类型的判断」，记住两个易错点：1\. 字符串无论内容是什么，只要用引号包裹，就是String类型；2\. typeof null 返回\&\#34;object\&\#34;，这是JS的设计缺陷，无需深究，记住即可。

### 练习题2：写出下列代码的运行结果，并说明原因

```js
// 题目
let num1 = 10;
let num2 = "10";
console.log(num1 == num2);
console.log(num1 === num2);
console.log(typeof (num1 + num2));
```

#### 答案：

true、false、\&\#34;string\&\#34;

#### 解析：

- num1 == num2：== 只比较“值”，会自动进行类型转换，把字符串\&\#34;10\&\#34;转成数字10，和num1（10）相等，所以返回true。

- num1 === num2：=== 比较“值 \+ 类型”，num1是Number类型，num2是String类型，类型不同，所以返回false（推荐日常开发用===，避免类型转换的坑）。
